The problem

Memory growth is invisible — until it isn't.

Leaks don't crash on commit. They accumulate. A controller here, a subscription there, a screen that quietly never lets go. Hours later your real-time app is sluggish and OOM-killed in the field — and the trace points at nothing.

DevTools can find it, if you know to go looking. Most teams don't — until a user does.

STATIC · custom_lint

Stop it before it ships

7 rules with auto-fixes flag the classics right in your editor — and they're false-positive-disciplined, so they understand idiomatic disposal.

  • › Undisposed controllers
  • › Uncancelled StreamSubscription & Timer
  • › Unclosed StreamController
  • › Discarded .listen() results & missing removeListener
  • › Bloc subscriptions never cancelled in close()
Understands if / try disposal, injected fields & locals.
RUNTIME · on-device

Catch what slips through

Inspects the live heap via the Dart VM service. Tracks per-class live-instance growth across forced-GC captures and flags classes that grow and never return.

  • › Draggable badge: live worst-severity + count
  • › Findings with sparklines & retaining paths
  • › Scan now, periodic, or auto on screen-pop
  • › Export findings as JSON / Markdown
Precise opt-in: track(obj) / markDisposed(obj).

What's inside

A toolkit, not a checkbox.

ZERO-CONFIG DETECTION

It tracks instance growth so you don't have to.

Per-class live-instance counts across forced-GC captures. Screens, blocs, controllers, subscriptions, timers — anything that grows and never returns gets flagged, automatically.

Real code

This is the whole integration.

main.dart 
// main.dart — one call, no base classes
import 'package:flutter_leak_radar/flutter_leak_radar.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await LeakRadar.init(LeakRadarConfig.standard(
    autoScan: const AutoScan(
      onNavigation: true,             // scan when a route pops
      period: Duration(seconds: 20),
    ),
    suspects: SuspectSet.defaults(),
  ));
  runApp(const _App());
}

// In your widget tree:
LeakRadar.overlay(child: const MyApp())
// NavigatorObserver:
navigatorObservers: [LeakRadar.navigatorObserver]

// optional precise opt-in — two calls, that's it
LeakRadar.track(controller, tag: 'ChatRoom');
LeakRadar.markDisposed(controller);
// chat_screen.dart
class _ChatScreenState extends State<ChatScreen> {
  final _controller = TextEditingController();

  Widget build(BuildContext context) => /* ... */;
}
